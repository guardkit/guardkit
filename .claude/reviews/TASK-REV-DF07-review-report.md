# Review Report: TASK-REV-DF07

## Executive Summary

Review of registering `python-library` and `nats-asyncio-service` as builtin templates in GuardKit. **The nats-asyncio-service template is near-ready for registration with minor fixes. The python-library template has critical metadata issues — its manifest and settings were incorrectly generated as JavaScript instead of Python — and requires significant cleanup before registration.**

Architecture Score: **45/100** (python-library drags the average; nats-asyncio-service alone would score ~72)

- **Mode**: Architectural Review
- **Depth**: Standard
- **Task**: TASK-REV-DF07
- **Reference Task**: TASK-REV-TI25 (langchain-deepagents-orchestrator registration — completed)

## Findings

### CRITICAL: python-library manifest.json is completely wrong

**Severity**: Critical | **Template**: python-library

The manifest.json was auto-generated by `/template-create` from the `youtube-transcript-mcp` source project, but the AI analysis **misidentified the language as JavaScript**:

| Field | Current (WRONG) | Should Be |
|-------|-----------------|-----------|
| `display_name` | "Javascript Layered Architecture" | "Python Library" |
| `description` | "JavaScript template using Layered Architecture architecture" | "Python pip-installable library with hatchling, src layout, pytest, ruff, mypy strict" |
| `language` | "JavaScript" | "Python" |
| `language_version` | null | ">=3.10" |
| `frameworks` | [] (empty) | hatchling, pytest, ruff, mypy |
| `architecture` | "Layered Architecture" | "Src Layout Library" |
| `category` | "general" | "library" |
| `tags` | JavaScript-related tags | Python library tags |

### CRITICAL: python-library settings.json uses JavaScript conventions

**Severity**: Critical | **Template**: python-library

Settings reflect TypeScript/JavaScript conventions instead of Python:

- `case_style`: "camelCase" for functions (should be "snake_case")
- `indent_size`: 2 (should be 4)
- `suffix`: ".ts" on classes (should be ".py")
- `layer_mappings`: Mix of "src/Infrastructure", "src/Shared" (JavaScript paths) and one correct `youtube_insights_mcp/services` (hardcoded source path)
- `file_patterns`: `["*.js", "*.jsx"]` (should be `["*.py"]`)

### CRITICAL: python-library CLAUDE.md says "youtube-transcript-mcp / JavaScript"

**Severity**: Critical | **Template**: python-library

The `.claude/CLAUDE.md` opens with:
```
# youtube-transcript-mcp
This is a JavaScript project using None.
Architecture: Layered Architecture
```

This must be rewritten to describe a generic Python library template.

### HIGH: python-library contains JavaScript artifacts

**Severity**: High | **Template**: python-library

The `templates/other/min/` directory contains 8 minified Lunr.js language files (`lunr.de.min.js.template`, `lunr.du.min.js.template`, etc.). These are artifacts from the source project's documentation site and have no place in a Python library template. They should be deleted entirely.

### HIGH: python-library agents are source-project-specific

**Severity**: High | **Template**: python-library

All 7 agent pairs are specific to the youtube-transcript-mcp source project:
- `youtube-transcript-specialist`
- `fastmcp-specialist` / `fastmcp-testing-specialist`
- `insight-extraction-specialist`
- `proxy-and-rate-limit-specialist`
- `cli-adapter-specialist`
- `pydantic-domain-model-specialist`

Per the template spec (`docs/research/dark_factory/template-spec-python-library.md`), MCP-specific patterns should be removed, and agents should be generalized for any Python library. The `pydantic-domain-model-specialist` could potentially be kept and renamed.

**Severity**: High | **Template**: python-library

`source_project`: `/Users/richardwoollcott/Projects/appmilla_github/youtube-transcript-mcp` — absolute path to author's machine. Should be removed or set to a public URL.

### MEDIUM: python-library confidence score is low (68.33)

### MEDIUM: nats-asyncio-service layer directory paths use PascalCase

**Severity**: Medium | **Template**: nats-asyncio-service

Settings.json layer_mappings use PascalCase directory names (`src/Entry Point`, `src/Application`, `src/Configuration`) which don't match Python conventions. Should be snake_case: `src/entry_point`, `src/handlers`, `src/services`, etc. — or better yet, match the actual template code structure (`{package_name}/handlers/`, `{package_name}/services/`).

### MEDIUM: nats-asyncio-service Author default is hardcoded

## Recommendations

### Recommendation 1: Regenerate python-library template (CRITICAL)

The python-library template needs to be regenerated or substantially reworked. The misidentification as JavaScript makes most metadata files unusable. Options:

**Option A (Recommended)**: Re-run `/template-create` against the source project with manual corrections, then apply the post-creation enhancements from the spec (remove MCP patterns, add py.typed, update category to "library", generalize agents).

**Option B**: Manual fix of manifest.json, settings.json, CLAUDE.md, remove JS artifacts, and replace/generalize agents. This is more work but avoids re-running the AI pipeline.

### Recommendation 2: Fix nats-asyncio-service metadata before registration

The nats-asyncio-service template is mostly correct. Fix:
- Layer directory paths → snake_case matching actual template structure
- Remove hardcoded `source_project` path
- Remove hardcoded `Author` default
- Fix `test_location` → "separate"
- Consider reducing complexity from 10 to 7-8

### Recommendation 3: Register nats-asyncio-service first, python-library second

Given the disparity in readiness:
1. Register `nats-asyncio-service` now (minor fixes only)
2. Fix `python-library` thoroughly, then register in a follow-up task

### Recommendation 4: Run `/agent-enhance --hybrid` after registration

Per the template specs, agents should be enhanced after registration to ensure they have full template context. Do this as a post-registration step for both templates.

### Recommendation 5: Update all registration points comprehensively

When updating init.py help text, also add the other missing templates (`react-fastapi-monorepo`, `mcp-typescript`, `fastmcp-python`) to keep the listing accurate.
