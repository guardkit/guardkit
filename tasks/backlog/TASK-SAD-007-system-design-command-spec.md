# Task: Write /system-design command specification

## Description

Write the complete command specification for `/system-design` at `installer/core/commands/system-design.md`. This command sits downstream of `/system-arch` and designs API contracts, data models, and multi-protocol surfaces per bounded context.

## Acceptance Criteria

- [ ] Prerequisite gate: check `has_architecture_context()` — if false, inform user and offer to chain to `/system-arch`
- [ ] Read bounded contexts and structural decisions from Graphiti (`project_architecture` group)
- [ ] Per-bounded-context interactive design flow:
  - API contract design (endpoints, request/response schemas, authentication)
  - Multi-protocol surface design (REST/GraphQL for web, MCP/A2A for agents, events for internal)
  - Data model design (entities, relationships, invariants)
  - Protocol selection: user chooses which protocols to support (MCP, A2A, ACP — ASSUM-008)
- [ ] Mandatory output artefacts:
  - API contracts per bounded context (using `api-contract.md.j2`)
  - OpenAPI 3.1 specification (`docs/design/openapi.yaml`)
  - MCP tool definitions (`docs/design/mcp-tools.json`) — if MCP selected
  - A2A task schemas (`docs/design/a2a-schemas.yaml`) — if A2A selected
  - Data model definitions with core entities and relationships
  - C4 Component diagrams per container (when >3 internal components OR explicitly requested)
  - Design Decision Records (DDRs) using `ddr.md.j2`
- [ ] Mandatory C4 L3 review gate: diagrams require explicit user approval
- [ ] OpenAPI validation quality gate: validate generated spec with `openapi-spec-validator`
- [ ] Graphiti seeding: all artefacts seeded into `project_design` and `api_contracts` groups
- [ ] Contradiction detection: flag proposed contracts that violate existing ADRs
- [ ] Graceful degradation when Graphiti unavailable
- [ ] DDR numbering: scan `docs/design/decisions/` for next available number
- [ ] Execution protocol section
- [ ] Error handling section

## Implementation Notes

- File: `installer/core/commands/system-design.md`
- Output directory: `docs/design/` (ASSUM-004 confirmed)
- DDR directory: `docs/design/decisions/` (parallel to ADR convention)
- OpenAPI 3.1 YAML generated by Claude, not Jinja2 templated (dynamic structure)
- MCP tool definitions and A2A schemas similarly generated directly by Claude
- Contradiction detection: query `project_decisions` group, compare proposed contract with ADR constraints
- C4 L3 threshold: >3 internal components triggers automatic L3 generation (ASSUM-012)
